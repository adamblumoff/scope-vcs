Implementation plan · July 14, 2026

Make workspace tabs feel instantaneous.

Replace route-wide reloads with a bounded, version-aware workspace cache; cancel stale work; resolve one file without rebuilding the repository; and stabilize the Code and History layouts. Ten or more open files should behave like an editor workspace, not ten repeated repository loads.

01

Decision and boundaries

This plan supersedes the workspace-tab loading strategy in the July 12 UI overhaul plan. The visual system and URL ownership remain intact.

Core decision. Open tabs become a bounded workspace cache, not a list of bookmarks. The URL remains canonical for the requested active path, while loaded data, request status, error state, and scroll position live in a keyed client workspace store.
Rendering decision. Keep only the active file or diff panel mounted. Retain data and explicit scroll state for at least ten open tabs. This avoids the memory and worker cost of ten mounted diff renderers while preserving instant reactivation.
Backend decision. A selected file must resolve through an indexed projection-file locator. Do not hydrate and reproject the complete repository merely to locate one path.
Cancellation decision. Selection is latest-wins. A → B → C immediately aborts A and B through every layer that supports cancellation; stale completions never update C’s panel.
Not the first fix. Do not begin with tab virtualization, extra animations, or mounting every tab panel. Ten tab elements are cheap. Repeated loaders, repository hydration, object reads, parsing, and layout reconstruction are expensive.
02

Delete the current hot path

There is no per-selection process or Git command to kill. The latency is duplicated application work across the browser, web server, API, database, and object store.

Current selectionOne click reloads repository-wide state.
  1. Write the tab ID to sessionStorage.
  2. Navigate the route search parameter.
  3. Reload the complete repository file list.
  4. Authenticate again through a second server function.
  5. Hydrate the full repository and recompute its projection.
  6. Read, decrypt, hash, copy, and JSON-serialize the blob.
  7. Rebuild the file tree and active content DOM.
→
Target selectionLocal first; one narrow cold-miss request.
  1. Set requestedId immediately and synchronize the URL.
  2. Read the workspace entry by immutable content identity.
  3. Warm hit: restore data and scroll state with zero requests.
  4. Cold miss: issue one direct, cancellable content request.
  5. Resolve one indexed projection-file locator.
  6. Read through immutable blob cache and singleflight.
  7. Commit only if the request generation is still current.
03

Workspace architecture

Separate stable collection data, active selection, cached resource data, and mounted UI. Each concern changes at a different frequency.

Stable collection

Repository tree

Owned above path selection and keyed by repository projection generation plus audience. Switching tabs does not reload it or change its object identity.

Immediate state

Requested selection

Updates synchronously on click. Drives active tab styling, pending indication, URL synchronization, and latest-wins request generation.

Bounded cache

Workspace entries

Stores status, immutable identity, data, error, scroll position, approximate bytes, and last access. LRU eviction prevents unbounded memory.

Render boundary

One mounted panel

The active file or diff is mounted. Hidden tabs retain data and view state, not expensive DOM and worker-owned renderers.

Server lookup

Projection locator

An indexed row provides path, OID, object key, SHA-256, size, and mode without hydrating the domain graph.

Immutable bytes

Blob cache

Small renderable blobs are cached by hash with singleflight. Mutation creates new keys rather than invalidating immutable content.

Target client workspace model
type WorkspaceEntry<T> = {
  id: string
  identity: string                 // repo generation + audience + path + OID
  status: 'idle' | 'loading' | 'loaded' | 'error'
  data?: T
  error?: string
  scrollTop: number
  approximateBytes: number
  lastAccessed: number
  requestGeneration?: number
}

type WorkspaceState<T> = {
  openIds: string[]
  requestedId: string | null       // optimistic visual selection
  settledId: string | null         // data currently shown
  entries: Map<string, WorkspaceEntry<T>>
}
Required selection state machine
  1. Save the current panel scroll position.
  2. Set the clicked tab as requestedId immediately.
  3. Open the tab once if it is not already present.
  4. Synchronize the URL without waiting for resource data.
  5. On a valid loaded entry, set settledId and restore scroll position.
  6. On a miss, abort the prior request, increment the generation, and start one request.
  7. Commit success or error only when both tab ID and request generation still match.
  8. Closing the active tab computes the next requested ID atomically; normalization must not resurrect the closed tab.

Cache contracts and invalidation

Use immutable identities whenever possible. Never cache private and public projections under the same key.

05

Implementation sequence

Land the user-visible latency fix before the deeper storage refactor. Every phase has a measurable exit gate and can be reviewed independently.

PR 1

Measure and protect

Establish failing performance and race tests before changing ownership.

Add deterministic fixtures and instrumentation for the current Code and History tab flows.

  • Count tree, content, history, commit-detail, and diff requests per activation.
  • Add delayed and out-of-order request controls for A → B → C.
  • Capture React commits, tree-row renders, layout-shift entries, and scroll offsets.
  • Cover active-tab close, final-tab close, and multiple rapid closes.
  • Record cold and warm traces with 10 files and a large repository fixture.
Exit gate: tests reproduce redundant collection requests, stale work, tab resurrection, and layout shift.
PR 2

Frontend lifecycle

Make warm Code-file switching local and immediate.

Separate the repository tree from selected-file loading and introduce the bounded workspace resource store.

  • Remove search.file from the stable tree resource dependency.
  • Add requestedId, settledId, resource entries, byte accounting, and LRU eviction.
  • Make selection optimistic; retain stale content visibly while a cold tab loads.
  • Pass the router abort signal through the selected-content client.
  • Persist only open-tab ordering in sessionStorage; keep file bodies in memory.
  • Save and restore scroll state explicitly by tab ID.
  • Partition workspace listeners by storage key.
Exit gate: cycling through ten loaded Code tabs performs zero network requests, zero tree rebuilds, and no incorrect scroll carryover.
PR 3

Stable layout and races

Remove visible jumps and deterministic close/select defects.

Decouple tab-strip geometry and panel layout from resource arrival.

  • Reserve the tab-strip row even when the workspace is empty or hydrating.
  • Compute active close + neighbor selection as one atomic workspace transition.
  • Move active-tab reveal into a useLayoutEffect keyed by activeId.
  • Use stable tab refs and avoid callback-ref layout reads on every strip render.
  • Keep History’s outer width and split tracks stable while opening or closing a diff.
  • Use compact parent suffixes within fixed tab widths for duplicate filenames.
Exit gate: repeated tab switching has approximately zero CLS; closing under delayed navigation never resurrects a tab.
PR 4

Indexed backend read

Turn a cold file load into one narrow metadata lookup plus one cached blob read.

Extend the projection read model so file content no longer requires repository hydration and projection recomputation.

  • Add object_key, sha256, size_bytes, and git_file_mode to projection-file rows.
  • Index the complete repository-version, source, audience, and path lookup.
  • Publish versioned projection rows atomically before marking a generation ready.
  • Replace hydrate_repo_from_row_id in the selected-file hot path.
  • Return text size_bytes so the browser deletes its full-string TextEncoder pass.
  • Add an immutable small-blob cache and per-hash singleflight.
Exit gate: a cold file request does not hydrate the repository or run project_graph; duplicate concurrent blob reads collapse to one object-store GET.
PR 5

History and large-content path

Apply the same cache model to diffs and remove main-thread stalls.

Separate history lists, commit detail, selected diff data, parsed diff state, and worker lifetime.

  • Cache history by repository/audience and commit detail by projected commit ID.
  • Cache diff data by commit, path, and immutable old/new OIDs.
  • Hoist the Pierre worker provider to a stable review-workbench boundary.
  • Move initial diff parsing off React’s render path or return server-computed hunks.
  • Use line-level diffing beyond measured file/line thresholds.
  • Virtualize large source or diff output only after profiling demonstrates the threshold.
  • Narrow live refresh so unrelated repository events do not reload the active content path.
Exit gate: warm diff switching is request-free and responsive; large diffs do not block input during the initial render.

The largest risks are stale authorization identities, cache memory growth, and fixing latency by retaining too much rendered state.

09

Definition of done

The work is complete only when the real app behaves like a stable reader workspace under rapid interaction.

  • Ten or more previously loaded Code files can be cycled without network requests or visible input lag.
  • Ten previously loaded History diffs can be cycled without reloading histories or commit detail.
  • File selection never reloads the complete repository tree.
  • A cold selected-file request does not hydrate or reproject the complete repository.
  • Rapid A → B → C selection aborts obsolete work and can never display A or B as C.
  • Closing an active tab is immediate and never temporarily restores the closed tab.
  • Closing the final tab performs no backend read and returns focus to the file navigator.
  • Each tab restores its explicit nested scroll position across source, README, binary, and diff modes.
  • Repeated switches produce approximately zero layout shift at desktop and mobile widths.
  • The tab strip reserves stable geometry and does not force layout reads for every attached tab ref.
  • Cache keys always include audience and immutable content identity.
  • Source and diff caches remain within measured byte/entry budgets after cycling 200 resources.
  • Concurrent requests for one immutable blob perform at most one object-store GET.
  • Large diffs do not synchronously block the main thread during React render.
  • Live repository events invalidate only affected generations/resources, not the entire matched route tree.
  • Keyboard tab navigation, close controls, focus restoration, reduced motion, and 44px coarse-pointer targets remain correct.
  • Automated checks pass and browser traces confirm the request, render, memory, and CLS budgets above.
Final user-visible standard. Opening a cold file may show a brief local pending state. Returning to any of the last ten or more loaded files must feel immediate, preserve reading position, and leave the surrounding workbench completely still.
