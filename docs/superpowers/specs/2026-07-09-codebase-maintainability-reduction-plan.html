Operating rules

Reduction without games.

Delete vertically. Remove model, persistence, API, CLI, UI, and tests together.

Keep one source of truth. Consolidate rules before deleting their copies.

Do not minify. Compressed, generated, macro-hidden, or unreadable code does not count.

Do not weaken coverage. Tests count only when redundant mechanics are removed.

No compatibility layer. This is pre-alpha; destructive schema resets are preferred.

Stop on behavior change. Any intended product change requires owner approval.

Prefer subtraction. New abstractions must delete more machinery than they add.

Verify each phase. Do not stack a later cleanup on an unverified earlier phase.

The LOC target is a constraint on architecture quality, not permission to erase meaningful tests, combine unrelated responsibilities, or replace clear code with clever code.

Postgres only

  • Delete memory-metadata, MemoryMetadataStore, and every memory/Postgres match branch.
  • Run local development and integration tests against disposable Postgres.
  • Keep pure domain tests on in-memory domain values without pretending they are a second database.

Native async API

  • Delete production MetadataStore::read and full-catalog hydration.

  • Use destructive pre-alpha reset rather than compatibility migration code.

7

Reap residual test machinery

Keep distinct regressions, delete false confidence, and make every fixture behavior-owned.

−5,509 Delivered net deletion

1. Remove misleading and zero-value tests first

  • Delete tests of API #[cfg(test)] Git credential, tree-import, and persistence shortcuts; replace necessary cases through production validation, promotion, authorization, or receive routes.
  • Delete trivial catalog construction, direct principal fields, clone/path interpolation, help-copy, default-folder, no-op wrapper, duplicated binary omission, and duplicated non-chunked first-push assertions.
  • Move the ignored Rust-to-TypeScript generator to an explicit tool or task. It remains counted source, but it stops pretending to be regression coverage.

2. Collapse by behavioral owner

  • Domain owns projection timelines, visibility canonicalization, request transitions, settlement, and side-effect declarations through compact scenario builders and decision tables.
  • Persistence owns mapping, transaction, lock, lease, generation-fence, and rollback behavior. Share isolated-schema setup and deterministic lock probes without hiding which aggregate a test exercises.
  • API owns translation, authentication, non-disclosure, protocol fidelity, and orchestration. Merge CLI/device auth matrices, request-ref servers, receive/config scenarios, visibility fixtures, and request workflow setup.
  • CLI owns command preflight, filesystem safety, credential scoping, and secret-free Git plans. Merge the two test-support layers and remove local Git-runner copies.
  • Web owns parsing and interaction coordination. Delete literal-template tests and transfer live-refresh coverage from trivial actor predicates to stale-event rejection, coalescing, retry, interruption, and cleanup.

3. Fix test infrastructure rather than serializing around it

  • Replace the process-global MemoryObjectStore with isolated test instances; do not rely on --test-threads=1 to mask cross-test interference.
  • Provide RAII Git servers, temporary repositories, and process coordination. Replace 25ms sleeps and one-second correctness deadlines with deterministic signals plus generous outer watchdogs.
  • Run the 11 local-dev tests in CI and wire the public web smoke test to a seeded CI environment; tests that never run do not count as protection.

4. Spend a small coverage budget on real gaps

  • Add one schema/reset introspection contract, one same-vs-different aggregate lock scenario, and compact outbox double-claim, expired-lease, and terminal-retry coverage.
  • Add one real cleanup generation-fence completion race and restore metadata-first deletion failure atomicity.
  • Add live-refresh coordinator coverage plus CLI credential get, push access matrix, and clone orchestration coverage.
  • Do not add per-DTO snapshots, happy-path copies at every layer, or new generalized fixture frameworks.

Risk and rollback

Sequence makes the campaign reversible.

Risk Control
Deleting behavior disguised as dead code Prove no production constructor/caller, then retain supported behavior characterization tests.
Memory/Postgres semantic drift during removal Make Postgres tests authoritative before deleting memory branches; compare ordering and error behavior.
Async conversion spreads too broadly Convert one persistence behavior family at a time; adapters await directly without sync wrappers.
Authorization drift Decision-table tests precede consolidation; API concealment tests remain at the boundary.
LOC pressure weakens readability Reject minification, giant files, hidden generated logic, or abstractions whose only purpose is reducing count.
Schema incompatibility Use explicit destructive pre-alpha reset; do not add compatibility migrations.
Each phase should land independently. If a phase changes supported behavior, stop and ask the owner rather than carrying an unapproved change into later refactors. Rollback is phase-level revert plus disposable database reset.
