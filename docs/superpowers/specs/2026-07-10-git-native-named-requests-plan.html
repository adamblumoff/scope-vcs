Git-native named requests

Product direction A clone or pull of the Scope origin should include every request available to that user. Requests should have required human names and appear directly as origin/<name>. There should be no join or sharing flow. A small scope pull command may handle authentication and provide clearer output, but normal Git should continue to work.

01

Decision

Requests are repository refs, not resources that users separately join or download.

Every request the current user can access is advertised by the Git remote as a normal branch.

Clone and pull transfer main plus those request branches in one upload-pack negotiation. REST continues to own request metadata and lifecycle; Git owns code and history transport.

02

03

Clone and pull flow

A single authenticated Git negotiation returns main and every visible request. There is no N+1 bundle fetch.

The public view is assembled without private objects. Hidden refs are not the security boundary; physically absent objects are.

Incremental pull
$ scope pull
Fetching origin as @public-contributor…
  [new request] railway-upload → origin/railway-upload
  [updated] auth-timeout   81a0e1c..9e73b24
Main is already up to date.

$ git switch --track origin/railway-upload
Switched to a new branch 'railway-upload'
scope pull is optional. It repairs Scope credential configuration when needed, runs a normal fast-forward-only Git pull, and summarizes request changes. A repository created with scope clone must also work with ordinary git fetch and git pull.

05

Data and API shape

The domain stores request identity and policy. The Git ref is derived; the filesystem read view remains disposable.

scope-core · request identity
struct Request {
  id: RequestId,          // req_…
  repo_id: RepoId,
  name: RequestName,      // railway-upload
  title: String,
  audience: Audience,     // Public | Private
  author_user_id: UserId,
  base_main_oid: GitOid,
  head_oid: GitOid,
  state: RequestState,
}

fn request_ref(name: &RequestName) -> String {
  format!("refs/heads/{name}")
}

08

Acceptance checks

The change is complete when plain Git accurately reflects the user’s Scope access.

Contributor clonePublic main and every public request arrive as origin/<name>; private objects are unreachable.
Maintainer clonePrivate main plus public and private request branches arrive in one negotiation.
Incremental pullNormal git pull discovers new requests and updates changed request refs.
Scope pull equivalencescope pull produces the same refs and adds only auth repair and readable output.
Named checkoutgit switch --track origin/railway-upload works without a join call.
Concurrent updateA stale public push is rejected until the contributor fetches and rebases.
Closed historyClosed request refs remain readable and reject all subsequent pushes.
Name safetyInvalid, duplicate, reserved, and ref-injection names fail before persistence.
